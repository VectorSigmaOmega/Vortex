================================================================================
PROJECT 1: VORTEX (MINIMALIST LINUX CONTAINER RUNTIME)
================================================================================
GOAL: Build a C++ binary that creates an isolated environment (jail) for a process.

CORE DELIVERABLES:
- A CLI tool: `./vortex run <rootfs_path> <command>`
- Process isolation (PID, Network, Mount, UTS namespaces).
- Resource limitation (Cgroups v2) for CPU/Memory.

TECHNICAL REQUIREMENTS:
- Syscalls: `clone()` with flags `CLONE_NEWPID | CLONE_NEWUTS | CLONE_NEWNS | CLONE_NEWNET`.
- Filesystem: Implementation of `pivot_root` and `mount("proc")`.
- Resource Control: Creating a directory in `/sys/fs/cgroup/` and writing the PID to `cgroup.procs`.
- Network: Basic Loopback interface setup inside the namespace.

IMPLEMENTATION PHASES:
1. Setup: Download an Alpine Linux Mini RootFS as the container image.
2. The Wrapper: C++ program that handles CLI arguments and calls `clone()`.
3. The Setup Function: The function executed by `clone()` that sets up hostname, mounts, and pivots the root.
4. The Cgroup Controller: Logic to write memory limits to the cgroup filesystem.
